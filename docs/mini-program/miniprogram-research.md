# 小程序开发调研文档

## 1. 项目概述

开发一个跨平台小程序，用于在移动端使用 AI No-Code 应用平台。

## 2. 调研结果

| 项目 | 选择 | 说明 |
|------|------|------|
| 开发框架 | Taro | 支持多平台编译 |
| 目标平台 | 微信、支付宝、抖音等 | 跨平台小程序 |
| UI风格 | 小程序原生风格 | 使用小程序原生组件 |
| 后端地址 | 本地 localhost | 开发阶段 |

| 功能 | 完整功能 | 浏览、创建、编辑、部署AI应用 |

## 3. 技术方案

### 3.1 框架选型 - Taro

Taro 是一个开放式跨端跨框架解决方案，支持使用 React/Vue 开发小程序。

- **优势**：一套代码编译到多平台（微信、支付宝、抖音、百度等）
- **网络请求**：使用 `Taro.request` 封装各平台原生 API
- **状态管理**：支持 MobX、Redux、轻量级状态管理

### 3.2 核心功能

| 功能 | 说明 |
|------|------|
| 用户登录 | 账号密码登录，保存 token |
| 应用浏览 | 查看精选/全部应用列表 |
| 应用详情 | 查看应用信息 |
| AI 对话 | 流式对话生成代码 |
| 应用预览 | 内嵌 webview 预览 |
| 版本管理 | 查看、切换应用版本 |
| 应用部署 | 一键部署到公网 |

### 3.3 API 对接

小程序调用后端 REST API，使用 `Taro.request` 封装网络请求：

```typescript
// 登录示例
Taro.request({
  url: 'http://localhost:8080/user/login',
  method: 'POST',
  data: { userAccount, userPassword },
})
```

**注意**：生产环境需要配置域名白名单或使用小程序转发功能。

### 3.4 登录流程

**阶段一：账号密码登录**
1. 用户输入账号密码
2. 调用 `/user/login` 获取登录态
3. 保存 token 到 storage
4. 后续请求带上 token（cookie 或 header）

**阶段二：微信登录（后期）**
1. 调用 `wx.login` 获取 code
2. 后端用 code 换取 openid/token
3. 实现静默登录
