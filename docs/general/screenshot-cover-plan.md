# Plan: Auto-Capture Preview Screenshot as App Cover Image (Server-Side)

## Context
Apps generated by AI are displayed on the HomePage as cards (`AppCard.vue`). Each `App` entity already has a `cover` field, and `AppCard` renders it as an image when present. Currently nothing populates this field. The goal is to **automatically capture a screenshot of the generated app's preview page using a server-side headless browser** after AI code generation completes, save it to the filesystem, and store the URL in `App.cover`.

### Design Principles
- **Interface-based**: `ScreenshotService` is an interface so the implementation can be swapped later (e.g., from local Chromium to an external API or OSS-based service).
- **Configurable**: The screenshot base URL is externalized to `application.yml`, not hardcoded.
- **Resilient**: Uses retry-with-backoff instead of fixed sleep delays.
- **Playwright for Java**: Reliable headless Chromium with built-in auto-wait and network-idle detection.
- **Storage**: Local filesystem for now (`tmp/user_code/covers/`). OSS migration planned — only `ScreenshotService` implementation needs to change.

### 2. Create `ScreenshotService` (interface + implementation)

**New file**: `src/main/java/fun/javierchen/ainocodeapplication/service/ScreenshotService.java`

Interface — allows swapping implementations later (local browser → external API → OSS service):
```java
public interface ScreenshotService {
    /**
     * Capture a screenshot of the given URL and save to outputFile.
     * @throws ScreenshotException if capture fails after retries
     */
    void captureScreenshot(String url, File outputFile);
}
```

**New file**: `src/main/java/fun/javierchen/ainocodeapplication/service/impl/PlaywrightScreenshotService.java`

Playwright-based implementation using bundled Chromium:
- Manages a Playwright `Browser` instance (lazy-initialized singleton, headless Chromium)
- Fixed viewport 1280×720 for consistent card thumbnails
- 10s page load timeout with `NETWORKIDLE` wait strategy
- Thread-safe via synchronized lazy init; each call gets its own `BrowserContext`/`Page`

```java
@Service
@Slf4j
public class PlaywrightScreenshotService implements ScreenshotService {
    private Playwright playwright;
    private Browser browser;

    private synchronized void ensureBrowserInitialized() {
        if (browser == null) {
            playwright = Playwright.create();
            browser = playwright.chromium().launch(
                new BrowserType.LaunchOptions().setHeadless(true)
            );
        }
    }

    @Override
    public void captureScreenshot(String url, File outputFile) {
        ensureBrowserInitialized();
        BrowserContext context = browser.newContext(
            new Browser.NewContextOptions().setViewportSize(1280, 720)
        );
        Page page = context.newPage();
        try {
            page.navigate(url, new Page.NavigateOptions()
                .setTimeout(10000)
                .setWaitUntil(WaitUntilState.NETWORKIDLE));
            page.screenshot(new Page.ScreenshotOptions()
                .setPath(outputFile.toPath())
                .setFullPage(false));
        } finally {
            page.close();
            context.close();
        }
    }

    @PreDestroy
    public void cleanup() {
        if (browser != null) {
            try { browser.close(); } catch (Exception e) {
                log.warn("Failed to close Playwright browser: {}", e.getMessage());
            }
        }
        if (playwright != null) {
            try { playwright.close(); } catch (Exception e) {
                log.warn("Failed to close Playwright: {}", e.getMessage());
            }
        }
    }
}
```

### 3. Add cover image serving endpoint  
**File**: [StaticResourceController.java](src/main/java/fun/javierchen/ainocodeapplication/controller/StaticResourceController.java)

Add `GET /static/covers/{filename}` endpoint **before** the catch-all `/{deployKey}/**` route (line 502):
- Validate filename: only allow pattern `\d+\.png` to prevent path traversal
- Serve from `PREVIEW_ROOT_DIR + "/covers/" + filename`
- Return with `Content-Type: image/png`

```java
@GetMapping("/covers/{filename}")
public ResponseEntity<Resource> serveCoverImage(@PathVariable String filename) {
    // Validate filename to prevent path traversal
    if (!filename.matches("\\d+\\.png")) {
        return ResponseEntity.badRequest().build();
    }
    
    File coverFile = new File(PREVIEW_ROOT_DIR + "/covers/" + filename);
    if (!coverFile.exists()) {
        return ResponseEntity.notFound().build();
    }
    
    Resource resource = new FileSystemResource(coverFile);
    return ResponseEntity.ok()
            .header("Content-Type", "image/png")
            .header("Cache-Control", "public, max-age=3600")
            .body(resource);
}
```

Add a private method with **retry-with-backoff** (replaces fragile `Thread.sleep(2000)`):
```java
private void captureAndSaveCoverAsync(Long appId, int version) {
    if (!screenshotEnabled) return;
    
    CompletableFuture.runAsync(() -> {
        String previewUrl = screenshotBaseUrl + "/api/static/preview/" + appId + "/" + version + "/";
        String coverDir = CodeFileConstant.CODE_FILE_PATH + File.separator + "covers";
        new File(coverDir).mkdirs();
        File coverFile = new File(coverDir, appId + ".png");
        
        // Retry with exponential backoff: 1s, 2s, 4s
        for (int attempt = 1; attempt <= maxRetries; attempt++) {
            try {
                long delay = 1000L * (1L << (attempt - 1)); // 1s, 2s, 4s
                Thread.sleep(delay);
                screenshotService.captureScreenshot(previewUrl, coverFile);
                
                // Update App.cover in database
                App updateApp = new App();
                updateApp.setId(appId);
                updateApp.setCover("/api/static/covers/" + appId + ".png");
                updateApp.setUpdateTime(LocalDateTime.now());
                appService.updateById(updateApp);
                
                log.info("App cover captured: appId={}, version={}, attempt={}", appId, version, attempt);
                return; // Success — exit retry loop
            } catch (Exception e) {
                log.warn("Screenshot attempt {}/{} failed for app {}: {}", attempt, maxRetries, appId, e.getMessage());
            }
        }
        log.warn("All {} screenshot attempts failed for app {}", maxRetries, appId);
    });
}
```

## Edge Cases
- **Screenshot fails** (browser crash, timeout): Retried up to `max-retries` times with exponential backoff. If all attempts fail, logged as warning, never blocks user flow. Cover stays empty, card shows default icon.
- **App not yet accessible** when screenshot runs: Retry-with-backoff (1s → 2s → 4s) handles the case where the preview isn't ready yet, much more robust than a fixed delay.
- **Overwriting previous cover**: Using `{appId}.png` filename naturally overwrites. Cache-busting `?t=timestamp` on frontend handles browser caching.
- **Concurrent generation**: Each app has its own cover file, no conflicts. Each screenshot call gets its own `BrowserContext`/`Page`, so concurrent captures are safe.
- **Playwright browser lifecycle**: Lazy-init singleton with `@PreDestroy` cleanup. If Chromium is not installed, all screenshots gracefully fail with warnings.
- **Vue project build delay**: For Vue projects, the screenshot runs after the build completes, so the dist directory should be ready. Retry-backoff provides additional safety margin.
- **Kill-switch**: Set `app.screenshot.enabled=false` in config to disable screenshot capture without code changes (useful for environments without Chromium).
- **Future OSS migration**: Only `ScreenshotService` implementation changes. `PlaywrightScreenshotService` → `OssScreenshotService` that uploads to cloud storage and returns a CDN URL.
