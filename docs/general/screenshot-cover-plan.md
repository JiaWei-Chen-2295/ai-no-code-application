# Plan: Auto-Capture Preview Screenshot as App Cover Image (Server-Side)

## Context
Apps generated by AI are displayed on the HomePage as cards (`AppCard.vue`). Each `App` entity already has a `cover` field, and `AppCard` renders it as an image when present. Currently nothing populates this field. The goal is to **automatically capture a screenshot of the generated app's preview page using a server-side headless browser (Playwright for Java)** after AI code generation completes, save it to the filesystem, and store the URL in `App.cover`.

### 2. Create `ScreenshotService`
**New file**: `src/main/java/fun/javierchen/ainocodeapplication/service/ScreenshotService.java`

A Spring `@Service` that:
- Manages a Playwright `Browser` instance (lazy-initialized singleton, headless Chromium)
- Exposes `captureScreenshot(String url, File outputFile)` method
- Opens a new page, navigates to the URL, waits for page load, takes a PNG screenshot
- Handles timeouts gracefully (max 10s wait)
- The screenshot dimensions should be a fixed viewport (e.g., 1280x720) for consistent card thumbnails

Key design:
```java
@Service
public class ScreenshotService {
    private Playwright playwright;
    private Browser browser;
    
    // Lazy initialization - only create browser when first screenshot is requested
    private synchronized void ensureBrowserInitialized() {
        if (browser == null) {
            playwright = Playwright.create();
            browser = playwright.chromium().launch(
                new BrowserType.LaunchOptions().setHeadless(true)
            );
        }
    }
    
    public void captureScreenshot(String url, File outputFile) {
        ensureBrowserInitialized();
        BrowserContext context = browser.newContext(
            new Browser.NewContextOptions().setViewportSize(1280, 720)
        );
        Page page = context.newPage();
        try {
            page.navigate(url, new Page.NavigateOptions()
                .setTimeout(10000)
                .setWaitUntil(WaitUntilState.NETWORKIDLE));
            page.screenshot(new Page.ScreenshotOptions()
                .setPath(outputFile.toPath())
                .setFullPage(false));
        } finally {
            page.close();
            context.close();
        }
    }
    
    @PreDestroy
    public void cleanup() {
        if (browser != null) browser.close();
        if (playwright != null) playwright.close();
    }
}
```

### 3. Add cover image serving endpoint  
**File**: [StaticResourceController.java](src/main/java/fun/javierchen/ainocodeapplication/controller/StaticResourceController.java)

Add `GET /static/covers/{filename}` endpoint **before** the catch-all `/{deployKey}/**` route (line 502):
- Validate filename: only allow pattern `\d+\.png` to prevent path traversal
- Serve from `PREVIEW_ROOT_DIR + "/covers/" + filename`
- Return with `Content-Type: image/png`

```java
@GetMapping("/covers/{filename}")
public ResponseEntity<Resource> serveCoverImage(@PathVariable String filename) {
    // Validate filename to prevent path traversal
    if (!filename.matches("\\d+\\.png")) {
        return ResponseEntity.badRequest().build();
    }
    
    File coverFile = new File(PREVIEW_ROOT_DIR + "/covers/" + filename);
    if (!coverFile.exists()) {
        return ResponseEntity.notFound().build();
    }
    
    Resource resource = new FileSystemResource(coverFile);
    return ResponseEntity.ok()
            .header("Content-Type", "image/png")
            .header("Cache-Control", "public, max-age=3600")
            .body(resource);
}
```

**For Vue projects** — in `generateAndSaveVueProjectCodeStream()` `doOnComplete` block (~line 194-201):

Add a private method:
```java
private void captureAndSaveCoverAsync(Long appId, int version) {
    CompletableFuture.runAsync(() -> {
        try {
            // Small delay to ensure the preview is ready to serve
            Thread.sleep(2000);
            
            String previewUrl = "http://localhost:8081/api/static/preview/" + appId + "/" + version + "/";
            String coverDir = CodeFileConstant.CODE_FILE_PATH + File.separator + "covers";
            new File(coverDir).mkdirs();
            File coverFile = new File(coverDir, appId + ".png");
            screenshotService.captureScreenshot(previewUrl, coverFile);
            
            // Update App.cover in database
            App updateApp = new App();
            updateApp.setId(appId);
            updateApp.setCover("/api/static/covers/" + appId + ".png");
            updateApp.setUpdateTime(LocalDateTime.now());
            appService.updateById(updateApp);
            
            log.info("App cover screenshot captured successfully: appId={}, version={}", appId, version);
        } catch (Exception e) {
            log.warn("Failed to capture cover screenshot for app {}: {}", appId, e.getMessage());
            // Never block the main flow
        }
    });
}
```

## Edge Cases
- **Screenshot fails** (browser crash, timeout): Logged as warning, never blocks user flow. Cover stays empty, card shows default icon.
- **App not yet accessible** when screenshot runs: The `captureAndSaveCoverAsync` uses the direct backend URL (`localhost:8081`) with a 2s delay, avoiding proxy issues. The preview is served by the same Spring Boot instance.
- **Overwriting previous cover**: Using `{appId}.png` filename naturally overwrites. Cache-busting `?t=timestamp` on frontend handles browser caching.
- **Concurrent generation**: Each app has its own cover file, no conflicts.
- **Playwright browser lifecycle**: Lazy-init singleton with `@PreDestroy` cleanup. If init fails (e.g., Chromium not installed), all screenshots gracefully fail with warnings.
- **Vue project build delay**: For Vue projects, the screenshot runs after the build completes, so the dist directory should be ready.
